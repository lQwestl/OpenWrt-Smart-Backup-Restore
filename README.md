# OpenWrt Smart Backup & Restore

Два скрипта для резервного копирования и восстановления конфигурации OpenWrt.  
Поддерживают **OpenWrt 24.x** (`opkg`) и **OpenWrt 25.x+** (`apk`), а также миграцию между версиями.

---

## Возможности

- Полный бэкап конфигурационных файлов и списка установленных пакетов
- Бэкап только списка пакетов (для установки на новом роутере)
- Восстановление бэкапа на ту же версию OpenWrt
- Миграция бэкапа с OpenWrt 24.x на 25.x без конфликтов
- Установка тем и дополнительных пакетов через меню
- Цветной вывод: зелёный — успех, жёлтый — предупреждение, красный — ошибка

## Установка

Скачиваем скрипт бэкапа
```sh
wget -O /root/smart_backup.sh https://raw.githubusercontent.com/lQwestl/OpenWrt-Smart-Backup-Restore/main/smart_backup.sh
wget -O /root/smart_restore.sh https://raw.githubusercontent.com/lQwestl/OpenWrt-Smart-Backup-Restore/main/smart_restore.sh
chmod +x /root/smart_backup.sh /root/smart_restore.sh
```

## Использование

### Создание бэкапа

```sh
./smart_backup.sh
```

**Меню:**
```
  1) Полный бэкап (конфиги + список пакетов)
  2) Бэкап только списка установленных пакетов
  3) Выход
```

**Пункт 1 — Полный бэкап** сохраняет:
- Все файлы из `/etc/config/`
- Изменённые конфигурационные файлы (через `opkg list-changed-conffiles` или `apk audit`)
- SSH-ключи хоста (`/etc/dropbear/`)
- SSL-сертификаты uHTTPd
- Ключи и конфигурацию репозиториев пакетного менеджера (`/etc/opkg/` или `/etc/apk/`)
- Crontabs, конфиги sing-box, правила nftables
- Системные файлы (`passwd`, `shadow`, `hosts`, `profile` и др.)
- Список установленных пакетов (`installed_packages.txt`)
- Метаданные бэкапа (`backup_metadata.txt`)

**Пункт 2 — Только список пакетов** сохраняет:
- `installed_packages.txt` — список пакетов для установки на новом роутере
- `backup_metadata.txt` — версия OpenWrt и пакетный менеджер

Бэкап сохраняется в `/root/backup/`.

---

### Восстановление

```sh
./smart_restore.sh
```

**Меню:**
```
  1) Восстановить бэкап (та же версия → та же версия)
  2) Миграция бэкапа (24.x → 25.x)
  3) Установить пакеты из сохранённого списка
  4) Установить тему Argon + Argon Config  (OpenWrt 24.x и 25.x)
  5) Установить тему Proton2025  (OpenWrt 24.x и 25.x)
  6) Установить Podkop
  7) Выход
```

---

#### Пункт 1 — Восстановление (та же версия)

Полное восстановление: конфиги, пакеты, репозитории.  
Используйте когда роутер сброшен к заводским настройкам или заменён на аналогичную модель с той же версией OpenWrt.

Порядок выполнения:
1. Выбор файла бэкапа из списка
2. Проверка интернет-соединения
3. Обновление списков пакетов и установка пакетов из списка
4. Восстановление конфигурационных файлов (включая конфиги репозиториев)
5. Финальный отчёт
6. Запуск сервисов *(SSH-соединение может прерваться)*

---

#### Пункт 2 — Миграция (24.x → 25.x)

Перенос конфигурации с роутера на OpenWrt 24.x (opkg) на роутер с OpenWrt 25.x (apk).

**Что исключается при миграции, чтобы не сломать систему:**
- `/etc/opkg/*.conf` — конфиги репозиториев opkg несовместимы с apk
- `/etc/config/firewall` — правила fw4 несовместимы между версиями и блокируют сеть
- `/etc/config/network` — форматы интерфейсов несовместимы (swconfig vs DSA, netmask vs CIDR)

После восстановления конфигов скрипт предложит ввести LAN IP-адрес для доступа к роутеру:

```
  При миграции с OpenWrt 24.x на 25.x конфигурация сети
  не переносится, так как форматы интерфейсов несовместимы.
  После перезапуска сети роутер будет доступен по IP-адресу,
  который вы укажете ниже.

  Введите IP-адрес для доступа к роутеру (например, 10.100.100.108):
```

---

#### Пункт 3 — Установить пакеты из списка

Устанавливает пакеты из ранее сохранённого `installed_packages.txt` на новый роутер.  
Удобно для переезда на новое устройство без полного восстановления конфигурации.

---

#### Пункт 4 — Argon Theme + Argon Config

Устанавливает тему Argon с русской локализацией на OpenWrt 24.x и 25.x через универсальный установщик.  
Для работы требуется `curl` — скрипт установит его автоматически, если он отсутствует.

Источник: [NerealNeSkill/luci-theme-argon-config-ru](https://github.com/NerealNeSkill/luci-theme-argon-config-ru)

---

#### Пункт 5 — Proton2025

Устанавливает тему Proton2025 на OpenWrt 24.x и 25.x.

Источник: [ChesterGoodiny/luci-theme-proton2025](https://github.com/ChesterGoodiny/luci-theme-proton2025)

---

#### Пункт 6 — Podkop

Устанавливает Podkop — инструмент для обхода блокировок.

Источник: [itdoginfo/podkop](https://github.com/itdoginfo/podkop)

> **Важно:** после установки Podkop необходимо создать профиль с именем **`main`** в настройках Podkop через веб-интерфейс LuCI. Без профиля `main` Podkop не будет работать.

## Что сохраняется в бэкапе

| Категория | Пути |
|-----------|------|
| Конфигурации UCI | `/etc/config/*` |
| SSH-ключи | `/etc/dropbear/dropbear_*_host_key` |
| SSL-сертификаты | `/etc/uhttpd.*` |
| Репозитории opkg | `/etc/opkg/keys/*`, `/etc/opkg/*.conf` |
| Репозитории apk | `/etc/apk/keys/*`, `/etc/apk/repositories.d/*`, `/etc/apk/repositories` |
| Crontabs | `/etc/crontabs/*` |
| Sing-box | `/etc/sing-box/*.json` |
| Правила nftables | `/etc/nftables.d/*.nft` |
| Системные файлы | `passwd`, `shadow`, `group`, `hosts`, `profile`, `rc.local`, `sysctl.conf` и др. |
| Список пакетов | `installed_packages.txt` |
| Метаданные | `backup_metadata.txt` |

---

## Требования

- OpenWrt 24.x (`opkg`) или OpenWrt 25.x+ (`apk`)
- Интернет-соединение для установки пакетов
- `curl` — только для установки Argon Theme (устанавливается автоматически)

---

## Заметки

- Бэкап сохраняется в `/root/backup/`
- Перед восстановлением скрипт всегда выводит предупреждение о перезаписи файлов
- Финальный отчёт выводится **до** перезапуска сервисов — SSH-соединение может прерваться при `network restart`
- При миграции 24→25 сетевой конфиг не переносится: нужно вручную ввести LAN IP в интерактивном запросе
